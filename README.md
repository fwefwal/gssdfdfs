### 2.3 Требования к данным  

**Основные сущности (бизнес-классы):**  

1. **Пользователь**  
   - ID (PK)  
   - Email  
   - Хэш пароля  
   - ФИО  
   - Телефон  
   - Роль (user, realtor, admin)  
   - Дата регистрации  

2. **Объект недвижимости**  
   - ID (PK)  
   - Тип (квартира, дом, участок, коммерция)  
   - Адрес (строка + геокоординаты)  
   - Площадь (общая/жилая)  
   - Кол-во комнат  
   - Этаж / этажность  
   - Цена  
   - Валюта  
   - Статус (продажа/аренда, активен/архив)  
   - Описание  
   - Дата публикации  
   - ID владельца (FK → Пользователь)  
   - ID модератора (FK → Пользователь, nullable)  
   - Статус модерации (на проверке, одобрено, отклонено)  

3. **Фото объекта**  
   - ID (PK)  
   - ID объекта (FK)  
   - URL файла  
   - Основное (bool)  

4. **Избранное**  
   - ID пользователя (FK)  
   - ID объекта (FK)  

5. **Заявка на просмотр**  
   - ID (PK)  
   - ID объекта (FK)  
   - ID пользователя (FK)  
   - Дата/время  
   - Комментарий  
   - Статус (новая, подтверждена, отменена)  

**Хранилище:** реляционная СУБД **PostgreSQL** с поддержкой PostGIS для геоданных.
